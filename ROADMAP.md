# ryon-project-command-tools 開発ロードマップ

このドキュメントは、本リポジトリに含まれるアプリ群の今後の開発方針と拡張計画をまとめたものです。

## 1. 全体方針

- **フロントエンド完結**: 引き続き、外部バックエンドAPIに依存しない静的Webアプリとして運用します。
- **データ永続化の段階的導入**: 現在は `localStorage` のみを使用していますが、必要に応じて IndexedDB や、ブラウザ完結のローカルファーストデータベースの導入を検討します。
- **PWA対応**: 学校現場やオフライン環境での利用を想定し、各アプリのPWA（Progressive Web App）化を進めます。

## 2. アプリ別ロードマップ

### 2.1 Today Work Selector
- **フェーズ1（完了）**: 27プロジェクトの静的データ化、推薦ロジック実装、プロンプト生成機能。
- **フェーズ2（短期）**: 
  - プロジェクトのカスタム追加・編集機能（UIからの追加）
  - 過去に選択したプロジェクトの履歴表示
- **フェーズ3（中期）**:
  - Notion API との連携（別リポジトリ `project-management-seed` との統合）による、プロジェクトデータの自動同期。

### 2.2 School Safety Checker
- **フェーズ1（完了）**: 初動チェックリスト、エリア管理、タイマー、ログ出力機能の実装。
- **フェーズ2（短期）**:
  - チェックリストと捜索エリアのカスタマイズ機能（学校ごとの設定保存）
  - ダークモード対応（夜間や暗所での使用を想定）
- **フェーズ3（中期）**:
  - PWA化による完全オフライン動作の保証
  - 複数端末間での状態共有（WebRTCやローカルネットワーク通信の実験的導入）

### 2.3 MaruFlow MVP Mock
- **フェーズ1（完了）**: MVP5画面のUIモック実装（架空データ）。
- **フェーズ2（短期）**:
  - ブラウザ内での簡易的な画像処理（Canvas/WebAssembly）による、採点枠の設定UIの高度化
  - ダミーデータではなく、ユーザーがアップロードしたPDF/画像をブラウザ内でプレビューする機能
- **フェーズ3（中期）**:
  - 実際のAI採点API（OpenAI Vision APIなど）との連携プロトタイプ開発
  - ※ APIキーを入力してブラウザから直接APIを叩くか、簡易的なBFF（Backend for Frontend）を用意するかは別途検討。

## 3. 新規アプリの構想

- **AI研修スライドジェネレーター**: 教員向けのAI活用研修スライドの構成案を、時間や対象者に応じて自動生成するツール。
- **授業計画JSONエディタ**: 単元計画や授業案をJSON形式で構造化して記述・管理するための専用エディタ。
