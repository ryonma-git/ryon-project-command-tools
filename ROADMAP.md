# ryon-project-command-tools 開発ロードマップ

このドキュメントは、本リポジトリに含まれる5つのアプリ群の今後の開発方針と拡張計画をまとめたものです。

## 1. 全体方針

- **フロントエンド完結**: 引き続き、外部バックエンドAPIに依存しない静的Webアプリとして運用します。
- **データ永続化の段階的導入**: 現在は `localStorage` のみを使用していますが、必要に応じて IndexedDB や、ブラウザ完結のローカルファーストデータベースの導入を検討します。
- **クロスプラットフォーム対応**: 当面はWebアプリとして構築しつつ、将来的にはiOS、macOS、Windowsのクロスプラットフォーム対応を目指します。

## 2. アプリ別ロードマップ

### 2.1 Today Work Selector
- **フェーズ1（完了）**: 27プロジェクトのデータ化、推薦ロジック、プロンプト生成、モード選択、お気に入り機能の実装。
- **フェーズ2（短期）**: 
  - プロジェクトのカスタム追加・編集機能（UIからの追加）
  - 過去に選択したプロジェクトの履歴表示
- **フェーズ3（中期）**:
  - Notion API との連携による、プロジェクトデータの自動同期。

### 2.2 School Safety Checker
- **フェーズ1（完了）**: 初動チェックリスト、エリア管理、タイマー、印刷対応、要再確認ステータス実装。
- **フェーズ2（短期）**:
  - チェックリストと捜索エリアのカスタマイズ機能（学校ごとの設定保存）
  - ダークモード対応（夜間や暗所での使用を想定）
- **フェーズ3（中期）**:
  - 複数端末間での状態共有（WebRTCやローカルネットワーク通信の実験的導入）

### 2.3 MaruFlow MVP Mock
- **フェーズ1（完了）**: 7画面のUIモック実装（統計・履歴を含む）。
- **フェーズ2（短期）**:
  - ブラウザ内での簡易的な画像処理（Canvas）による、採点枠の設定UIの高度化
  - ユーザーがアップロードしたPDF/画像をブラウザ内でプレビューする機能
- **フェーズ3（中期）**:
  - 実際のAI採点API（OpenAI Vision APIなど）との連携プロトタイプ開発

### 2.4 School Ops Portal
- **フェーズ1（完了）**: UIモック実装、記事一覧・検索・プロンプト生成。
- **フェーズ2（短期）**:
  - Markdownエディタによる新規記事の作成・編集機能
  - 画像やファイルの添付機能（ブラウザ内保存）
- **フェーズ3（中期）**:
  - ローカルファーストDBを用いた本格的なナレッジベース化

### 2.5 Manazashi Archive Demo
- **フェーズ1（完了）**: デモ画面実装、時系列・子ども別表示。
- **フェーズ2（短期）**:
  - 実際の記録アプリとしての入力UI実装
  - 写真・動画の添付機能
- **フェーズ3（中期）**:
  - 音声入力による記録作成支援（Web Speech API連携）

## 3. 新規アプリの構想

- **AI研修スライドジェネレーター**: 教員向けのAI活用研修スライドの構成案を、時間や対象者に応じて自動生成するツール。
- **授業計画JSONエディタ**: 単元計画や授業案をJSON形式で構造化して記述・管理するための専用エディタ。
