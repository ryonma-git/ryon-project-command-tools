# App Specifications (アプリ仕様書まとめ)

本リポジトリに含まれる5つのアプリの基本仕様をまとめます。各アプリの詳細な設計は個別の設計書（`docs/*-design.md` または `docs/*.md`）を参照してください。

## 1. Today Work Selector
- **目的**: 27のプロジェクトから、現在の状況（時間、気力、場所、ツール）に最適なタスクを推薦する。
- **主要機能**: 
  - 8つのWorkMode（おまかせ、校務ops、personal-memory等）。
  - お気に入り機能。
  - ChatGPT/Claude向けプロンプト自動生成。
- **データ**: `src/data/projects.ts` に静的定義。

## 2. School Safety Checker
- **目的**: 学校での緊急事案発生時の初動対応における抜け漏れを防ぐ。
- **主要機能**: 
  - 5種の事案テンプレート。
  - カテゴリ別チェックリスト（優先度バッジ付き）。
  - 捜索エリア管理（要再確認ステータス）。
  - 印刷、Markdownコピー、JSONエクスポート。
- **制約**: 判断の自動化ではない。個人情報を保存させない。

## 3. MaruFlow MVP Mock
- **目的**: 紙テスト前提の採点支援アプリ「MaruFlow」のUIモックアップ。
- **主要機能**: 
  - ホーム、PDF登録、採点欄設定、採点、出力確認、統計、履歴の7画面。
- **制約**: 実データ処理やAI OCRは未実装。

## 4. School Ops Portal
- **目的**: 学校ICT・校務ナレッジの管理とプロンプト生成。
- **主要機能**: 
  - カテゴリ・タグによる絞り込み。
  - 全文検索。
  - ナレッジに基づくChatGPTプロンプト生成。
- **データ**: `src/data/articles.ts` に静的定義。

## 5. Manazashi Archive Demo
- **目的**: 子どもの「まなざし」（記録）を時系列で整理し、「学びの物語」として出力するデモ。
- **主要機能**: 
  - 児童3名分の6年タイムライン。
  - 教師の観察メモ、本人の振り返り表示。
  - 3スタイル（やさしい文体、卒業文集風、保護者向け）での物語生成。
- **制約**: 架空データのみ使用。所見自動生成や成績管理ではない。
