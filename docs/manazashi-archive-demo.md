# Manazashi Archive Demo 設計・仕様書

Manazashi Archive Demo（まなざしアーカイブ架空デモ）は、学校における子どもの学びの記録（教師の観察メモ、本人の振り返り、発言など）を時系列で整理し、「学びの物語」として出力するコンセプトデモアプリです。

## アプリの目的と位置づけ

- **目的**: 「学校に残る先生のまなざしを、子ども自身の人生に返す」というコンセプトを、架空データを用いて体験・検討すること。
- **位置づけ**: 成績管理や所見の自動生成、児童の監視を目的としたものではありません。あくまで「記録の整理」と「物語としての振り返り」を補助するツールです。

## 主要機能

### 1. 架空データの提示
実在の児童データを使用せず、1〜6年生までの厚みのある架空データ（児童3名分）を用意しています。
- 各児童には、理科、英語、図工、生活科、友人関係など、多様な場面での記録が含まれます。
- 教師の観察メモに加え、児童本人の振り返りや発言（引用）も記録されています。

### 2. タイムライン表示
6年間の学びの記録を時系列で表示します。
- **フィルタリング**: 学年、教科、タグによる絞り込みが可能。
- **視覚的表現**: 教科ごとのカラーリングや、発言・振り返りのアイコン表示により、直感的に記録を振り返ることができます。

### 3. 「あなたの学びの物語」生成
蓄積された記録をもとに、3つの異なるスタイルで「学びの物語」を出力します。

- **やさしい文体**: 本人への語りかけ。温かく親しみやすい言葉。主な読者は本人（子ども）。
- **卒業文集風**: 少し改まった文体。6年間を振り返る。主な読者は本人・クラスメート。
- **保護者向け説明文風**: 客観的な説明文体。成長を整理して伝える。主な読者は保護者・教師。

### 4. 倫理的配慮の明示
アプリ内に「倫理説明ページ」を設け、以下の点を明記しています。
- 所見の自動生成ではないこと。
- 成績管理ではないこと。
- 監視ではないこと。
- 判断の自動化ではないこと。
- 実データ投入前に必要な検討事項（保護者の同意、個人情報保護、データ管理など）。

## データ構造（types.ts）

```typescript
export interface LearningRecord {
  grade: number;           // 学年（1〜6）
  term: string;            // 学期・時期
  subject: string;         // 教科・活動
  event: string;           // 出来事・活動内容
  quote?: string;          // 印象的な発言
  workMemo?: string;       // 作品メモ
  teacherNote: string;     // 教師の観察メモ
  selfReflection?: string; // 本人の振り返り
  tags: string[];          // キーワードタグ
}

export interface FictionalChild {
  id: string;
  alias: string;           // 架空の呼称（「児童A」など）
  type: string;            // 学びのタイプ説明
  profile: string;         // 架空プロフィール
  records: LearningRecord[];
}
```
