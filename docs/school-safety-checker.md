# School Safety Checker 設計・仕様書

School Safety Checker は、学校における緊急事案（校内外の所在不明、けが、不審者、ICT障害など）発生時の初動対応を支援するチェックリスト・ログ記録アプリです。

## アプリの目的と位置づけ

- **目的**: 緊急時の初動対応における「確認漏れ」や「連絡漏れ」を防ぎ、対応記録を時系列で残すこと。
- **位置づけ**: 判断を自動化するものではなく、教職員の判断と行動を補助するツールです。アプリの各画面には「これは判断の自動化ではありません。最終判断は教職員が行います」という免責事項を明記しています。

## 主要機能

### 1. 事案テンプレートの選択
発生した事案の性質に応じて、最適なチェックリストと対応手順を読み込みます。

- **校内所在不明**: 校舎内での児童生徒の行方不明事案。
- **校外所在不明**: 登下校中や校外学習での行方不明事案。
- **けが・事故**: 救急搬送や保護者連絡を伴うけが。
- **不審者**: 学校周辺での不審者情報や侵入事案。
- **ICT障害**: ネットワークダウンや端末の広範なトラブル。

### 2. カテゴリ別チェックリスト
対応タスクをフェーズ・目的別に分類し、優先度（`critical`, `high`）を視覚的に表示します。

- **共有**: 管理職への報告、全職員への周知。
- **確認**: 事実関係の把握、安全確認。
- **捜索**: 捜索エリアの分担と状況確認。
- **連絡**: 保護者、警察、消防、教育委員会への連絡。
- **記録**: 対応ログの作成。
- **事後対応**: 事案収束後のケアや再発防止。

### 3. 捜索エリア管理
捜索場所ごとのステータスを管理し、重複や漏れを防ぎます。
- **ステータス**: 未確認 → 捜索中 → 確認済 → 要再確認

### 4. 経過時間タイマー
事案発生からの経過時間をリアルタイムで表示し、時間経過に応じた次のアクション（例：30分経過で警察へ連絡など）の判断基準を提供します。

### 5. 記録の出力・エクスポート
対応ログを様々な形式で出力し、事後報告や引き継ぎに活用できます。

- **Markdown出力**: 報告書の下書きとして利用可能。
- **JSONエクスポート**: システム連携やデータ保存用。
- **印刷プレビュー**: ブラウザの印刷機能に最適化されたCSS（`@media print`）を適用。不要なUI要素（ボタンやナビゲーション）を非表示にし、ログとチェックリストをきれいに印刷します。

### 6. データ保存と個人情報保護
- **localStorage保存**: ブラウザを閉じてもデータを保持する機能（ON/OFF切替可能）。
- **個人情報注意**: 保存を有効にする際、「実名などの個人情報を入力しないよう」注意喚起モーダルを表示します。

## 技術スタック

- React 18
- TypeScript
- Vite
- CSS Modules (純粋なCSS)

## 倫理的配慮

- アプリはあくまで「補助ツール」であり、教職員の専門的判断を代替するものではありません。
- 個人情報の入力を最小限に抑えるよう、UI上で継続的に注意喚起を行っています。
- 緊急時のパニックを防ぐため、UIはシンプルで直感的な操作性を重視しています。
